# 用户故事

## 1. 角色定义

| 角色 | 描述 |
| --- | --- |
| 普通开发者 | 使用 RepoPilot 生成代码修改、查看测试结果并发起 PR |
| 仓库维护者 | 管理已接入仓库、审批 Agent 生成的修改 |
| 系统管理员 | 配置模型、GitHub token、Docker 沙箱和系统参数 |
| 面试展示者 | 使用标准 Demo 展示项目亮点和技术闭环 |

## 2. 用户故事列表

### US-001 用户注册登录

作为普通开发者，我希望能注册和登录系统，以便管理自己的仓库和 Agent 任务。

验收条件：

- 用户可通过邮箱和密码注册。
- 登录成功后返回 JWT。
- 未登录访问项目、任务和审批接口时返回 401。

### US-002 添加 GitHub 仓库

作为普通开发者，我希望输入 GitHub 仓库地址和 token 后接入项目，以便让系统分析代码。

验收条件：

- 系统校验仓库 URL 格式。
- clone 成功后记录默认分支、最新 commit、仓库本地路径。
- clone 失败时展示可读错误原因。

### US-003 查看项目结构

作为普通开发者，我希望查看仓库文件树和 Spring Boot 模块识别结果，以便确认系统理解了项目。

验收条件：

- 能看到文件树。
- 能看到 Controller、Service、Mapper、Entity 的数量。
- 能查看每个符号对应的文件路径。

### US-004 创建 Agent 任务

作为普通开发者，我希望用自然语言创建任务，以便让 Agent 协助开发或修复 bug。

验收条件：

- 任务必须绑定项目。
- 支持 FEATURE、BUGFIX、REVIEW、DOC 四种任务类型。
- 创建后状态为 `CREATED`。

### US-005 查看 Agent 执行过程

作为普通开发者，我希望看到 Agent 每一步执行过程，以便判断结果是否可信。

验收条件：

- 能看到 Planner、Retriever、Coder、Tester 等步骤。
- 每个步骤展示状态、开始时间、结束时间和摘要。
- 失败步骤展示错误原因。

### US-006 预览 diff

作为仓库维护者，我希望在应用修改前预览 diff，以便确认 Agent 没有做危险改动。

验收条件：

- diff 使用 unified diff 格式保存。
- UI 或 API 能按文件展示变更。
- diff 未审批前不能创建 PR。

### US-007 沙箱测试

作为仓库维护者，我希望 Agent 生成代码后自动运行测试，以便降低错误 PR 的概率。

验收条件：

- 系统在隔离环境执行 `mvn test`。
- 保存退出码、日志、耗时。
- 测试失败时 Agent 总结失败原因。

### US-008 人工审批

作为仓库维护者，我希望确认或拒绝 Agent 的修改，以便保持最终控制权。

验收条件：

- Approve 后 patch 状态变为 `APPROVED`。
- Reject 后任务不能继续创建 PR。
- Regenerate 会创建新的 Agent run 和 patch 版本，并重新执行沙箱测试后回到待审批状态。

### US-009 创建 Pull Request

作为仓库维护者，我希望审批后自动创建 GitHub PR，以便进入标准代码审查流程。

验收条件：

- 系统创建目标分支。
- 系统提交修改并推送。
- 系统创建 PR 并返回 PR URL。

### US-010 查看工具调用审计

作为系统管理员，我希望查看 Agent 调用了哪些工具，以便排查失败和控制风险。

验收条件：

- 每次工具调用记录工具名、输入摘要、输出摘要、状态和耗时。
- 敏感字段必须脱敏。
- 任务详情能关联所有工具调用。
